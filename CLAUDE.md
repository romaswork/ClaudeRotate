# ClaudeRotate

## Что это
Приложение для macOS на SwiftUI с иконкой в Dock и значком в меню-баре. Назначение —
ротация учётных данных Claude API в JSON-файле конфигурации (формат Claude Code
`settings.json`) по таймеру. Пользователь ведёт список именованных ключей, может
включать/выключать каждый, а приложение по очереди подставляет включённые ключи
в целевой файл каждые N минут. Все данные хранятся локально.

## Поведение ротации
Поддерживаются **две независимые цели**, у каждой свой переключатель
включено/выключено и свой выбранный файл. При ротации текущий ключ пишется во все
**включённые и выбранные** цели; если включены обе — ключ меняется в обоих файлах
одновременно (значение `APIKey.apiKey` одно и то же).

**Цель Claude Code** (`settings.json`). Меняются ТОЛЬКО следующие поля, остальное
содержимое сохраняется:
- `apiKeyHelper` → `echo '<ANTHROPIC_API_KEY>'` (формируется автоматически из ключа)
- `env.ANTHROPIC_API_KEY` → значение ключа
- `env.ANTHROPIC_BASE_URL` → **эффективный** базовый URL ключа: собственный
  `baseURL` ключа, а если он пуст у ключа категории FreeModel — общий
  `freeModelBaseURL` категории (`AppStore.effectiveBaseURL(for:)`)
- `env.HTTPS_PROXY` / `env.HTTP_PROXY` → URL прокси, привязанного к ключу. Если
  прокси не назначен, эти поля **удаляются** из `env` (чтобы убрать прокси,
  оставшийся от предыдущего ключа).

**Цель Codex** (`auth.json`, формат `{ "OPENAI_API_KEY": "<ключ>" }`, по умолчанию
`~/.codex/auth.json`). Меняется ТОЛЬКО поле `OPENAI_API_KEY` → значение ключа;
прокси к Codex не применяется. Остальное содержимое сохраняется.

Файл читается через `JSONSerialization`, изменяются перечисленные поля, записывается
обратно с `[.prettyPrinted, .sortedKeys]`; экранирование слешей убирается. Все прочие
ключи и значения сохраняются (порядок ключей может нормализоваться).

Запись **неатомарная** (`Data.write(to:)` без `.atomic`). Причина: под App Sandbox
bookmark даёт доступ только к самому файлу, но не к его каталогу, а `.atomic` создаёт
временный файл рядом и делает rename — для этого нужен доступ к директории. Файл
крошечный, запись занимает доли мс, риск повреждения при сбое минимален.

## Ключевые решения
- **Песочница (App Sandbox) включена** (`ENABLE_APP_SANDBOX = YES`, entitlements в
  `KeyRotator/KeyRotator.entitlements`) — требование Mac App Store. Доступ к
  каждому целевому файлу — только через **security-scoped bookmark**: пользователь
  выбирает файл вручную (`NSOpenPanel`), приложение хранит bookmark
  (`AppStore.fileBookmark` для Claude, `codexFileBookmark` для Codex, оба app-scope)
  и перед каждым чтением/записью открывает доступ через `AppStore.withTargetAccess`
  (Claude) / `withCodexAccess` (Codex). Произвольный ввод пути текстом не
  поддерживается; `filePath`/`codexFilePath` — только для отображения. При обновлении
  со старой (несендбокс) версии bookmark отсутствует — пользователь должен один раз
  заново выбрать файл.
  Entitlements: `app-sandbox`, `network.client` (проверка ключей и прокси),
  `files.user-selected.read-write`, `files.bookmarks.app-scope`.
- **Иконка в Dock + значок в меню-баре** — флаг `LSUIElement` не задан, приложение
  по умолчанию отображается и в Dock, и в трее. Видимость в Dock переключается в
  рантайме настройкой «Скрывать из Dock» (`hideFromDock`) через
  `NSApp.setActivationPolicy(.accessory/.regular)` — без перезапуска.
  Значок в меню-баре — **собственный `NSStatusItem`** (`StatusItemController`),
  а не SwiftUI `MenuBarExtra`: `MenuBarExtra` не различает кнопки мыши, а нужно,
  чтобы **левый клик открывал главное окно** приложения, а **правый (или
  Ctrl+клик) показывал контекстное меню** (меню собирается заново при каждом
  показе — строки всегда актуальны).
- `apiKeyHelper` формируется автоматически из API-ключа.
- **Категории ключей** — у ключа есть `category` (`KeyCategory`: `general` /
  `freeModel`). FreeModel — ключи, полученные через одноимённый сервис; в боковой
  панели они живут в подменю «FreeModel» под пунктом «Ключи» (`DisclosureGroup`).
  У категории FreeModel есть **общий Base URL** (`AppStore.freeModelBaseURL`,
  редактируется на вкладке «FreeModel» настроек); ключ с собственным
  `baseURL` переопределяет общий. Резолв — `effectiveBaseURL(for:)`, он же
  используется при записи в файл, при проверке ключа и в отображении (в списке
  унаследованный URL показывается с пометкой «Общий:»). Ротация идёт по всем
  включённым ключам обеих категорий по порядку общего списка `keys`. Старые
  конфиги без поля `category` декодируются как `general`.
- **Лимиты аккаунта FreeModel** — сервис лимитирует расход двумя скользящими
  окнами: 5 часов и 7 дней. Данные берутся запросом
  `GET https://freemodel.dev/api/usage` с cookie `bm_session` (токен сессии
  дашборда, живёт ~30 дней; пользователь копирует его из браузера в поле «Токен
  сессии» редактора FreeModel-ключа, хранится в `APIKey.usageToken`). По самому
  API-ключу (`fe_oa_…`) данные недоступны: эндпоинт авторизуется только сессией,
  а `cc.freemodel.dev` не отдаёт rate-limit-заголовков. Формат ответа:
  `window5h`/`windowWeek` = `{usedCents, limitCents, resetsAt}` (unix-секунды).
  Запросы дедуплицируются по токену (у ключей одного аккаунта токен общий —
  один запрос на аккаунт, результат раскладывается по всем его ключам).
  Запрос идёт **сначала через прокси**, привязанный к любому из ключей группы
  токена (глобальный переключатель `proxiesEnabled` намеренно игнорируется —
  он управляет только записью в целевой файл); если через прокси не удалось
  (транспортная ошибка, 407 и любой HTTP-код кроме 401/403) — повторяется
  напрямую (фолбэк внутри `fetchFreeModelUsage`). Автообновление (управляется
  настройкой `freeModelAutoRefresh`, вкладка «FreeModel» в настройках): при
  запуске сразу, далее таймер-тикер каждые 30 с запускает «просроченные»
  группы — аккаунт **активного** ключа обновляется раз в
  `freeModelActiveRefreshMinutes` (по умолчанию 2 мин), остальные — раз в
  `freeModelOthersRefreshMinutes` (по умолчанию 15 мин); при выключенном
  автообновлении фоновых запросов нет (тикер продолжает идти и подхватывает
  включение без перезапуска). Дополнительно: при входе в раздел FreeModel (не
  чаще раза в минуту), при смене токена в редакторе, кнопкой в каждой строке
  и кнопкой «Обновить все» в нижнем тулбаре
  списка FreeModel (`refreshAllUsageSequentially` — аккаунты опрашиваются по
  очереди с настраиваемой паузой `freeModelSequentialPauseSeconds` после
  каждого ответа, по умолчанию 3 с). Отображение — в строках списка FreeModel:
  base URL в строке не показывается, вместо него всю свободную ширину строки
  занимает панель лимитов — два индикатора «5 ч»/«7 дн» (растянутая полоса
  прогресса, процент, суммы «$46.19 / $66» и обратный отсчёт до сброса
  «через 1 ч 12 м», тикающий раз в минуту через `TimelineView`; точная дата
  сброса и время обновления — в тултипе) плюс кнопка обновления. Цвет
  заполнения — по четвертям (`FreeModelUsage.Window.tint`): <25% зелёный,
  25–50% жёлтый, 50–75% оранжевый, ≥75% красный; когда активен FreeModel-ключ
  с загруженными лимитами (и включена настройка `freeModelMenuBarIcon`
  «Лимиты в меню-баре»), значок в меню-баре заменяется кастомной иконкой:
  сверху полоска этого цвета (окно 5 ч активного ключа), под ней процент
  использования. **Автопереключение
  при исчерпании**: когда любое из окон лимитов (5 ч или 7 дн) активного
  FreeModel-ключа доходит до
  порога `freeModelSwitchThresholdPercent` (50–100%, по умолчанию 100%),
  проигрывается системный звук (настройки `freeModelSoundEnabled` /
  `freeModelSoundName`, по умолчанию «Glass», фолбэк — beep;
  `AppStore.playExhaustSound()`, он же — прослушивание в настройках) и — если
  включена настройка `freeModelAutoSwitch` — активным становится **верхний
  ключ таблицы FreeModel** в её текущем порядке
  (`AppStore.displayedFreeModelKeys()` — автосортировка по лимитам или ручной
  порядок), кроме самого исчерпанного и кандидатов, у которых любое окно уже
  за порогом (`AppStore.isKeyExhausted`; ключи без загруженных лимитов не
  пропускаются); если подходящих включённых FreeModel-ключей нет, смена не
  происходит (звук всё равно играет).
  Детект — в `AppStore.performUsageFetch` по **переходу** через
  порог (раньше ниже порога или данных не было, стало ≥ порога; предикат —
  `AppStore.isUsageExhausted`), поэтому когда
  исчерпаны все ключи, повторных переключений на каждом опросе нет;
  переключение — через хук `AppStore.onActiveKeyExhausted`, который
  `RotationManager` в `init` привязывает к `switchFromExhaustedKey()`
  (работает и при остановленной ротации).
  Список FreeModel
  **автосортируется по лимитам** (только отображение, порядок ротации — по
  общему списку `keys`): сначала по заполнению 5-часового окна (0% → 100%),
  при равенстве — по моменту сброса 7-дневного окна (чей сброс раньше, тот
  выше); ключи без загруженных лимитов — после них в исходном порядке, а
  полностью исчерпанные (любое окно на 100%, независимо от порога
  переключения) — в самом конце.
  Автосортировку можно выключить настройкой `freeModelAutoSort` — тогда
  порядок ручной (перетаскиванием, как у обычных ключей); при включённой
  автосортировке перетаскивание в списке FreeModel отключено.
  Токен в целевой файл не записывается, но входит в экспорт (внутри `keys`).
- **Настройки FreeModel** — все параметры категории собраны на отдельной
  вкладке «FreeModel» раздела «Настройки» (вкладка «Общие» — остальные
  настройки; переключатель — segmented-пикер вверху `SettingsView`). Десять
  персистируемых свойств `AppStore` (входят в `Config`, экспорт/импорт и
  `resetAll`): `freeModelAutoRefresh` (вкл/выкл автообновления лимитов),
  `freeModelActiveRefreshMinutes` (1–60, интервал активного аккаунта),
  `freeModelOthersRefreshMinutes` (1–180, интервал остальных),
  `freeModelSequentialPauseSeconds` (0–60, пауза «Обновить все»),
  `freeModelAutoSwitch` (автопереключение при исчерпании),
  `freeModelSwitchThresholdPercent` (50–100, порог исчерпания),
  `freeModelSoundEnabled` / `freeModelSoundName` (звук исчерпания; выбор из
  списка системных звуков, при выборе звук проигрывается для прослушивания),
  `freeModelMenuBarIcon` (кастомная иконка с процентом в меню-баре),
  `freeModelAutoSort` (автосортировка списка по лимитам). Здесь же —
  единственное место редактирования общего Base URL категории
  (`freeModelBaseURL`).
- **Локализация интерфейса** (RU/EN) — не через `.strings`/`.lproj`, а через
  `AppStore.tr(ru, en)`: строки задаются прямо в коде парами, выбранный язык хранится
  в конфиге (`language`), смена применяется мгновенно без перезапуска (т.к. `language`
  — `@Published`, любой вызов `tr` в `body` перерисовывается). Все новые строки UI
  обязательно оборачивать в `store.tr(...)`.
- **Прокси** — отдельный список именованных прокси (с авторизацией или без). Ключу
  можно привязать прокси (`APIKey.proxyID`); при ротации URL прокcи пишется в
  `env.HTTPS_PROXY`/`env.HTTP_PROXY`. URL прокси: `http://[user:pass@]host[:port]`,
  логин/пароль percent-кодируются. При удалении прокси он автоматически отвязывается
  от всех ключей.

## Структура проекта
- `KeyRotator/APIKey.swift` — модель ключа (id, name, apiKey, baseURL, enabled,
  proxyID — привязанный прокси, category — категория `KeyCategory`
  `general`/`freeModel`, usageToken — опциональный токен сессии дашборда
  FreeModel для показа лимитов; кастомный декодер подставляет `general` для
  старых конфигов без поля).
- `KeyRotator/FreeModelUsage.swift` — модель и загрузка лимитов FreeModel:
  `FreeModelUsage` (`window5h`/`windowWeek`, у окна `usedCents`/`limitCents`/
  `resetsAt` + вычисляемые `fraction`, `resetDate`; расширение `Window.tint` —
  квартильный цвет заполнения зелёный/жёлтый/оранжевый/красный, общий для
  списка и иконки меню-бара), UI-состояние
  `FreeModelUsageState` (loading/loaded(at:)/failure) и
  `fetchFreeModelUsage(sessionToken:proxy:)` — `GET https://freemodel.dev/api/usage`
  с cookie `bm_session` в эфемерной `URLSession` без автоматических cookie
  (чтобы токены разных аккаунтов не смешивались); 401/403 = токен истёк.
  При переданном прокси запрос сначала идёт через него (авторизация — общий
  `ProxyAuthDelegate` из ProxyTester.swift), при неудаче — фолбэк напрямую;
  ответы `ok`/`unauthorized` через прокси напрямую не перепроверяются.
- `KeyRotator/Proxy.swift` — модель прокси (id, name, host, port, username, password),
  вычисляемые `url` (с percent-кодированием логина/пароля), `copyString` (строка
  `логин:пароль@host:port` без `http://` для копирования в буфер) и `displayName`.
- `KeyRotator/ProxyTester.swift` — `testProxy(_:)`: проверка прокси запросом к
  сервису геолокации (`ipwho.is`, HTTPS) через него. Возвращает `ProxyTestResult`
  с задержкой (ping) и страной выхода (`ProxyCheck`: latencyMs, countryCode → флаг и
  локализованное имя через `Locale`, ip). `407` = ошибка авторизации (через делегат
  сессии `ProxyAuthDelegate` — internal, переиспользуется загрузкой лимитов
  FreeModel); транспортная ошибка = недоступен. UI-состояние — `ProxyTestState`.
- `KeyRotator/AppLanguage.swift` — перечисление языка интерфейса (`russian`/`english`),
  `displayName`, `systemDefault` (по локали системы).
- `KeyRotator/KeyRotator.entitlements` — entitlements песочницы (см. «Ключевые решения»).
- `KeyRotator/AppStore.swift` — единый источник данных (`ObservableObject`),
  хранение конфигурации в `~/Library/Application Support/ClaudeRotate/config.json`, CRUD
  ключей и прокси (`proxy(for:)` резолвит прокси ключа), локализация через `tr(ru, en)`.
  Категории: `freeModelBaseURL` (общий Base URL FreeModel, персистится и входит в
  экспорт), `keys(in:)`, `sortedFreeModelKeys()` (автосортировка таблицы FreeModel:
  заполнение 5-часового окна 0%→100%, затем более ранний сброс 7-дневного
  окна; без данных — после них, полностью исчерпанные (любое окно ≥100%) — в
  самом конце), `displayedFreeModelKeys()` (порядок таблицы с
  учётом настройки `freeModelAutoSort`: автосортировка или ручной порядок;
  используется списком и автопереключением), `isUsageExhausted(_:)` /
  `isKeyExhausted(_:)` (любое окно ≥ порога `freeModelSwitchThresholdPercent` —
  общий предикат детекта исчерпания и фильтра кандидатов автопереключения),
  `move(in:from:to:)` (перестановка внутри категории — слоты
  ключей других категорий не меняются), `testAll(in:)` и `effectiveBaseURL(for:)`
  (собственный baseURL ключа или общий FreeModel-URL при пустом; используется при
  записи и проверке). Настройки FreeModel — десять персистируемых
  `@Published`-свойств `freeModel…` (см. «Настройки FreeModel» выше) +
  `playExhaustSound()` (звук исчерпания по `freeModelSoundName`, фолбэк beep).
  Лимиты FreeModel: `usageStates` (`[UUID: FreeModelUsageState]`, транзиентно),
  `refreshAllUsage(minInterval:)` / `refreshUsage(for:)` / `hasUsageTokens`;
  запросы группируются по токену (`usageTokenGroups`), защищены от дублей
  (`usageTokensInFlight`) и троттлятся (`usageFetchedAt`); в запрос передаётся
  первый прокси, привязанный к ключам группы (`assignedProxy`, глобальный
  переключатель прокси игнорируется). Фоновый таймер-тикер (30 с) стартует в
  `init` и через `autoRefreshUsageTick` (no-op при выключенном
  `freeModelAutoRefresh`) обновляет группу активного ключа раз в
  `freeModelActiveRefreshMinutes`, остальные — раз в
  `freeModelOthersRefreshMinutes`; `refreshAllUsageSequentially` (кнопка
  «Обновить все») опрашивает аккаунты по очереди с паузой
  `freeModelSequentialPauseSeconds` после ответа,
  флаг выполнения — `usageRefreshingAll` (`@Published`, транзиентно).
  `performUsageFetch` после записи состояний детектирует переход любого из
  окон лимитов (5 ч / 7 дн) активного ключа через порог
  `freeModelSwitchThresholdPercent` — играет
  звук (если `freeModelSoundEnabled`) и, если включён `freeModelAutoSwitch`,
  зовёт хук `onActiveKeyExhausted` (транзиентный, устанавливает
  `RotationManager`).
  CRUD ключей подтягивает/чистит лимиты
  (add/updateKey при смене токена, deleteKey; importData/resetAll сбрасывают).
  Две независимые цели (Claude/Codex) с флагами `claudeEnabled`/`codexEnabled` и
  путями `filePath`/`codexFilePath`. Доступ к целевым файлам: `setTargetFile(_:)` /
  `setCodexFile(_:)` (создают bookmark из выбранного URL), `withTargetAccess(_:)` /
  `withCodexAccess(_:)` (общий приватный `withAccess` резолвит bookmark, открывает
  security-scoped доступ, обновляет устаревший bookmark), `hasTargetFile` /
  `hasCodexFile` и `hasAnyActiveTarget` (включена ли хоть одна цель с файлом).
  Импорт/экспорт/сброс:
  `exportData()` (сериализует ключи, прокси, интервал, автозапуск, язык в JSON;
  bookmark и путь к целевому файлу НЕ включаются — доступ специфичен для машины),
  `importData(_:)` (заменяет настройки данными из файла, сбрасывает рантайм-состояние),
  `resetAll()` (очищает все ключи, прокси и настройки до значений по умолчанию и
  забывает целевой файл; сам settings.json пользователя не трогается).
- `KeyRotator/RotationEngine.swift` — `writeKey(_:baseURL:proxy:to:)` (запись ключа,
  уже резолвнутого эффективного base URL и прокси в Claude-файл) и
  `writeCodexKey(_:to:)` (запись только `OPENAI_API_KEY` в
  Codex-файл), оба пишут в уже резолвнутый security-scoped URL неатомарно;
  `RotationManager` (таймер, `start`/`stop`/`rotateNow`/`rotatePrevious`/

## Сборка
xcodebuild требует полный Xcode (не Command Line Tools), поэтому указываем
`DEVELOPER_DIR`:

```
DEVELOPER_DIR=/Applications/Xcode.app/Contents/Developer \
  xcodebuild -project ClaudeRotate.xcodeproj -scheme KeyRotator \
  -configuration Debug -destination 'platform=macOS' build
```

Собранное приложение: `~/Library/Developer/Xcode/DerivedData/ClaudeRotate-*/Build/Products/Debug/KeyRotator.app`.

## Технические детали
- macOS deployment target: 14.0, Swift 5.0, изоляция по умолчанию — MainActor.
  Планку держат API уровня macOS 14: `ContentUnavailableView`/`.search`,
  двухпараметрический `.onChange(of:) { _, _ in }`, `.background(.background.secondary, in:)`.
- Bundle ID: `com.romas.clauderotatekey`. Таргет, папка исходников (`KeyRotator/`) и
  имя приложения (`CFBundleName`/`CFBundleDisplayName`/`PRODUCT_NAME`) — **KeyRotator**
  (нейтральное, без товарного знака «Claude»; совместимость с Claude Code указывается
